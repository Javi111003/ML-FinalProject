### 1. Introduction

The **Cuban Telecommunications Company S.A. (ETECSA)** provided a dataset for the purpose of developing various studies and models based on **Machine Learning techniques**.
These data reflect the use of different telecommunication services by customers, such as **phone calls, text messages (SMS), balance recharges, and mobile data consumption**, among others.

The main objective of this analysis is to **understand the structure, content, and characteristics** of the dataset, in preparation for its use in predictive models and behavioral analysis.

---

### 2. General Description of the Dataset

The dataset is organized in a tabular format containing **10,000 records** and **40 variables**, distributed across columns that describe different aspects of each transaction or service usage event.

Each row represents a **Call Detail Record (CDR)**, documenting an event generated by a customer, such as making a call, sending a text message, or connecting to the mobile network.

The following table summarizes the most relevant types of variables:

| Type of Variable              | Example Fields                                           | General Description                                                                                                |
| ----------------------------- | -------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------ |
| **Identifiers**               | `CDR_ID`, `OBJ_ID`, `OWNER_CUST_ID`                      | Uniquely identify each record, object, or associated customer.                                                     |
| **Temporal**                  | `START_DATE`, `END_DATE`                                 | Indicate the start and end date/time of the service used.                                                          |
| **Categorical**               | `SERVICE_CATEGORY`, `FLOW_TYPE`, `USAGE_SERVICE_TYPE`    | Specify the type of service, its category (voice, data, SMS, recharge), and traffic direction (incoming/outgoing). |
| **Numerical**                 | `ACTUAL_USAGE`, `ACTUAL_CHARGE`, `TOTAL_TAX_AMOUNT`      | Measure service usage (e.g., minutes, megabytes, messages) and the associated monetary charges.                    |
| **Nested Structures / Lists** | `CHARGE_LIST`, `CHARGE_SERVICE_INFO`, `BALANCE_CHG_LIST` | Describe charge details, taxes, and balance modifications for each event.                                          |

These fields are complemented by auxiliary information such as measurement units, account identifiers, billing cycles, and reserved columns for future system expansions.

### 5. Potential Machine Learning Applications

The ETECSA dataset offers a wide range of analytical and predictive possibilities, including:

1. **Customer Behavior Analysis:**
   Identify usage patterns based on frequency, service type, or average spending.

2. **User Segmentation:**
   Classify customers into groups (clusters) according to their consumption habits or service preferences.

3. **Demand and Usage Prediction:**
   Estimate future service consumption (minutes, data, SMS) based on historical patterns.

4. **Anomaly or Fraud Detection Models:**
   Detect irregular behaviors or inconsistencies in billing and service usage records.

5. **Churn Prediction Models:**
   Predict the likelihood that a user will discontinue a given service.

All analyzed data were **anonymized** prior to use, removing any personally identifiable information such as phone numbers or customer details.
This ensures **compliance with privacy and confidentiality standards**, and guarantees that the analysis focuses exclusively on general and technical service usage patterns.
